SHIPPING ESTIMATOR APPLICATION
TESTING & EXECUTION GUIDE
============================================================

1. PREREQUISITES
------------------------------------------------------------

Before running the application, ensure the following:

- Java 21 or above installed
- Maven installed
- IDE (VS Code / IntelliJ / Eclipse)
- Postman (for API testing)

To check Java:
> java -version

To check Maven:
> mvn -version


2. HOW TO RUN THE APPLICATION
------------------------------------------------------------

Method 1: Using IDE

1. Open project in IDE.
2. Navigate to ShippingEstimatorApplication.java
3. Right click → Run.
4. Wait until you see: "Started ShippingEstimatorApplication"

Application will run on:
http://localhost:8080


Method 2: Using Maven Command Line

In project root directory:

> mvn clean install
> mvn spring-boot:run

Application runs on:
http://localhost:8080


3. DATABASE ACCESS (H2 Console)
------------------------------------------------------------

Open browser:
http://localhost:8080/h2-console

JDBC URL:
jdbc:h2:mem:testdb

Username: sa
Password: (leave empty)

Click Connect to view tables.


4. API TESTING USING POSTMAN
============================================================

A. NEAREST WAREHOUSE API
------------------------------------------------------------

Method: GET

URL:
http://localhost:8080/api/v1/warehouse/nearest?sellerId=1&productId=1

Expected Output:
- Returns nearest warehouse details
- Status: 200 OK


B. SHIPPING CHARGE API
------------------------------------------------------------

Method: GET

URL:
http://localhost:8080/api/v1/shipping-charge?warehouseId=1&customerId=1&productId=1&deliverySpeed=standard

Expected Output:
- Calculates distance using Haversine formula
- Applies transport rate based on distance
- Adds standard courier charge
- Returns total shipping charge
- Status: 200 OK


C. EXPRESS SHIPPING TEST
------------------------------------------------------------

Change deliverySpeed to express:

http://localhost:8080/api/v1/shipping-charge?warehouseId=1&customerId=1&productId=1&deliverySpeed=express

Expected Output:
- Applies express surcharge (₹1.2 per kg)
- Returns higher value than standard shipping
- Status: 200 OK


D. COMBINED SHIPPING API
------------------------------------------------------------

Method: POST

URL:
http://localhost:8080/api/v1/shipping-charge/calculate

Body → Raw → JSON:

{
  "sellerId": 1,
  "customerId": 1,
  "productId": 1,
  "deliverySpeed": "standard"
}

Expected Output:

{
  "shippingCharge": 180.00,
  "nearestWarehouse": {
    "warehouseId": 1,
    "warehouseLocation": {
      "lat": 12.99999,
      "lng": 37.923273
    }
  }
}


5. ERROR HANDLING TESTS
============================================================

A. Invalid Warehouse ID
------------------------------------------------------------

Use warehouseId = 999

Expected Output:
Status: 404
Message: "Warehouse not found"


B. Product Not Belonging To Seller
------------------------------------------------------------

Use mismatched sellerId and productId.

Expected Output:
Proper validation error message.

- Strategy pattern for delivery type
- Global exception handling

- Clean layered architecture
